\b SXMaterialEditor – редактор материаловдля движка SkyXEngine. Основное окно (также как и окно рендера) не изменяемо в размерах. \image material_editor.files/image1.png "Скришот программы SXMaterialEditor"

 

Окно состоит из структурных элементов (сверху вниз):

Заголовок окна, содержащий имя редактор и имя материала.

Меню программы:

File:

Open – открыть материал

Save – сохранить материал

Exit – выход из программы

Final image (выводимое изображение в окно рендера):

            Color – цвет

            Normals – нормали

            Parameters – параметры освещения

            Ambient diffuse – фоновое и диффузное освещение

            Specular – отраженный свет

Lighting scene – освещенная сцена

Панель инструментов, где расположены основные управляющие кнопки:

 открыть материал

 сохранить материал

 возможность перемещения камеры

 установка камере старновой позиции и поворота

 вывод на рендер цвета

 вывод на рендер нормалей

 вывод на рендер параметров освещения

 вывод на рендер фонового и диффузного освещения

 вывод на рендер отраженного света

 вывод на рендер освещенной сцены

Содержимое окна – рабочая область по настройке материала.

 

Управление камерой:

CTRL + w – движение вперед

CTRL + s – движение назад

CTRL + a – движение влево

CTRL + d – движение вправо

SHIFT - ускорение

CTRL + ПКМ – движение вверх/вниз

CTRL + ЛКМ – вращение

 

Общее

 кнопка выбора файлов, после нажатия на эту кнопку следует открытие диалога выбора файла

 поле для ввода цифровой/текстовой информации, после ввода при нажатии на клавишу Enter введенные значения будут приняты. Также изменения вступят в силу после того как этот элемент потеряет фокус (к примеру щелкнув на пустое место/на другой элемент этого же окна)

Все текстуры (в том числе и текстуры скайбокса) должны находится по относительному пути (относительно запущенной программы) по адресу: /gamesource/textures/

Все шейдеры должны находится по относительному пути (относительно запущенной программы) по адресу: /gamesource/shaders/

 

 

Редакторные настройки

 вращение модели по оси Y, кнопка 0 установит угол вращения в 0

 

 загрузка кубической текстуры скайбокса

 

 выбор типа модели, от выбора зависит рендер и передаваемые данные (принимаемые шейдером), доступно:

 тестовые модели:

sphere

Базовые настройки

 основная текстура материала

 набор комбинаций шейдеров (для упрощения выбора шейдеров), если активна нулевая строка (пустая) значит, выбрана комбинация которой нет в списке

 вершинный шейдер

 пиксельный шейдер

 физический материал:

concrete (бетон)

metal (металл)

glass (стекло)

plastic (пластик)

tree (дерево)

flesh (плоть)

ground/sand (земля/песок)

water (вода)

leaf/grass (листва/трава)

 

 коэффициент пробиваемости [0,1]

 нужно ли освещать материал

 использовать ли альфа тест (отсечение пикселей которые не прошли минимальный порог прозрачности)

 тип преломления:

none – нет преломления

transparency lighting – освещаемый полупрозрачный материал

 

 тип отражения:

none – нет отражения

reflect plane – плоское отражение

reflect cube dynamic – динамическое объемное/кубическое отражение

reflect cube static - статическое объемное/кубическое отражение, обновляется только в случае изменения данных самого отражения

 

 

Параметры освещения

 набор параметров освещения (для упрощения настройки освещения), если активна нулевая строка (пустая) значит выбранных параметров нет в списке

 просвечиваемость, коэфициент самозатенения (для листвы и травы надо делать меньше 1) [0,1]

 шероховатость поверхности, от этого зависит рассеиваемость света по поверхности, чем больше тем больше будет рассеивание света [0,1]

 отражательная способность поверхности, чем больше тем меньше будет поглощено света и тем больше будет его отражение [0,1]

 текстура с параметрами освещения (где можно настроить для каждого пикселя, каналы red – roughness, green - f0, blue – thickness), если стоит галочка на texture то будет использована загруженная текстура с параметрами освещения, если текстура не загружена будут использоваться нулевые параметры, если галочка не стоит – будут использоваться параметры из редактора

 детальная текстура (четвертая), при использовании шейдером наложения по маске - коэффициент яркости данной текстуры в alpha канале маски

Наборы данных

Поддерживаемые наборы данных:

пары шейдеров (расположение относительно программы shaders_kit.cfg)

параметры освещения (расположение относительно программы paraml_kit.cfg)

 

shaders_kit.cfg

Файл конфигураций должен содержать секцию shaders_kit с ключом count в котором храниться количество секций с наборами шейдеров в данном файле.

Каждая секция именуется как shader_порядковыйномер, нумерация с нуля. Каждая такая секция обязательно должна содержать ключ name с именем данного набора, а также ключи:

vs содержищий имя вершинного шейдера с расширением

ps содержащий имя пиксельного шейдера с расширением.

 

paraml_kit.cfg

Файл конфигураций должен содержать секцию paraml_kit с ключом count в котором храниться количество секций с параметрами освщеения в данном файле.

Каждая секция именуется как paraml_порядковыйномер, нумерация с нуля.            Каждая такая секция обязательно должна содержать ключ name с именем набора, а также ключи

thickness – float [0,1] параметр просвечиваемости

roughness – float [0,1] параметр шероховатости

f0 – float [0,1] параметр отражательной способности поверхности
