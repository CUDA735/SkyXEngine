Управление полупрозрачными поверхностями

Каждый выводимый пиксель помечается номером поверхности к которой он относится

  • 0 - пусто
  • 1 - непрозрачная геометрия
  • все что больше 1 значит полупрозрачная геометрия
    Заметки
    Если инкремент установлен то отрисовка каждого материала будет инкрементировать внутренний счетчик, присваивая каждому пикселю поверхности идентификатор поверхности.
    На данный момент максимальное значение полупрозрачных поверхностей может быть только 255 (8 бит)

◆ MTL_TEX_R_DETAIL

#define MTL_TEX_R_DETAIL   2

#include <sxmtllight.h>

стартовый регистр для детальных текстур

Предупреждения
4 следующих (включая указанный) будует заполнены детальными текстурами, поэтому никакие другие текстуры в [MTL_TEX_R_DETAIL, MTL_TEX_R_DETAIL+3] не должны отправляться
Заметки
Текстуры для каждого канала будут отправляться последовательно: r, g, b, a - MTL_TEX_R_DETAIL, MTL_TEX_R_DETAIL + 1, MTL_TEX_R_DETAIL + 2, MTL_TEX_R_DETAIL + 3

◆ SML_MtlClear()

SX_LIB_API void SML_MtlClear ( bool  clear_ref_del)

#include <sxmtllight.h>

очитска, удаляет все загруженные материалы (но место в массиве под них остается), все стандартные остаются

Предупреждения
Отражения не удаляются а помещаются в очередь удаления, так как могут сожеражть в себе идентификаторы, которые необходимы очистить, для полной очистки необходимо вызывать функцию SML_MtlDelRefClear. Однако во избежания утечек памяти необходимо очистить идентификаторы, для их получения вызыва функцию SML_MtlDelRefGetIDArr
Аргументы
clear_ref_delочищать ли очередь удаления отражений, на тот случай если идентификаторы в отражениях больше не действительны
