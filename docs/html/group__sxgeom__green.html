Массивы просчетов видимости

Предупреждения
Каждый вызов SGeom_GreenAddArrForCom создает новый элемент массива просчета, который занимает определенное количества памяти, которое зависит от объема текущего списка растительности и их делений.
Поэтому без причинные вызовы данной функции не допустимы.
Если идентификатор созданный SGeom_GreenAddArrForCom больше не актуален (это определяет только программист) то его необходимо освободить память которую он занимает через функцию SGeom_GreenDelArrForCom
Идентификаторы возвращаемые SGeom_GreenAddArrForCom константны, и не будут удалены или сдвинуты пока сам программист этого не сделает, либо в случае перезагрузки

Подробное описание

Заметки
Рендер растительности основан на hardware instancing
Рендер единицы растительности происходит на основании кусков, рендер единичной модели исключен, поэтому некоторые эффекты присущие идинчным экзмеплярам рендера будут ошибочные (к примеру отражения)
В любой момент времени, при наличии в загруженных моделей, можно пройти по списку моделей циклом, начиная с 0 и до SGeom_GreenGetCount
Пример распаралеливания просчетов видимости и рендера:
ID id_arr_cam = SGeom_GreenAddArrForCom();
ID id_arr_shadow = SGeom_GreenAddArrForCom();
ID id_arr_light = SGeom_GreenAddArrForCom();
//...
SGeom_GreenComVisible(frustum_cam,pos_cam,id_arr_cam);
SGeom_GreenComVisible(frustum_shadow,pos_shadow,id_arr_shadow);
SGeom_GreenComVisible(frustum_light,pos_light,id_arr_light);
//...
SGeom_GreenRender(timeDelta, viewpos, type, id_arr_cam);
SGeom_GreenRender(timeDelta, viewpos, type, id_arr_shadow);
SGeom_GreenRender(timeDelta, viewpos, type, id_arr_light);
Необходимо сделать:

Описать формат файла с сохраненным списком растительности

Добавить возможность единичного добавления в массив растительности (мышкой тыкаем в окне где должно быть дерево например и оно там появляется)

Добавить возможность единичного удаления из массива растительности

)

#include <sxgeom.h>

добавить единицу растительности

Заметки
path, lod1, lod2 - пути относительно стандартного пути, до модели и лодов, если lod1 и lod2 равны 0, тогда будет добавлен элемент трава, иначе элемент дерево. Лоды рисуются на определенных расстояниях
При генерации происходит проверка на столкновение растительности со всей геометрией (lock/unlock буферов ... другого способа не нашли)
Информация о месте генерации растительности берется из маски текстуры path_mask (которая должна быть сохранена в dds формат A8R8G8B8) на основании значения альфа канала. Количество генерируемых объектов в текущем пикселе поределяется по формуле:
//alpha от 0 - 255
//count_max - аргумент функции #SGeom_GreenAddGreen
int count = int(floor((alpha*count_max * 100) / 2550.f));
В целом текстура представляет из себя весь уровень по ширине и длине, таким образом каждый пиксель текстуры это определенное место на уровне:
//bbmin и bbmax - точки экстремума, ограничивающий параллелепипед всей геометрии
//x_tex и y_tex - координаты текущего пикселя в текстуре
//width_tex и height_tex - ширина и высота текстуры
//просчет позиции только по оси xz
float2 pixel_position = lerpf(bbmin.xz, bbmax.xz, float2(x_tex + 1, y_tex + 1) / float2(width_tex,height_tex));
и описывает определенный периметр:
//width_level и depth_level - ширина и длина всей геометрии
//x_tex и y_tex шинира и высота текстуры в пикселях
float2 pixel_perimeter = float2(width_level,length_level) / float2(x_tex,y_tex);
Таким образом чем больше текстура в пикселях размера уровня в метрах тем меньший периметр описывает один пиксель и тем большую плотность можно сделать и наоборот.
При генерации деревьев идет проверка на отсутсвие других деревьев в радиусе вписанной окружности bound box'а дерева по оси xz
Предупреждения
Возвращаемый идентификатор не является константным. При удалении другой единицы, которая в списке находится выше, идентификатор текущей будет смещен! Иными словами: id растительности не постоянен, если удалить растительность из списка то все элементы после id станут на единицу меньше
