Подробное описание

Заметки
Распознавание текстур (обращение через функции) происходит на основе идентификаторов представленных типом ID.
Имя текстуры обязательно долно содержать нижний пробел (_), строка до первого нижнего проблема это имя папки в котором находится текстура с целым именем.
Например: mtl_tex.dds - лежит по загружаемому пути: /mtl/mtl_tex.dds

Макросы

◆ SXGC_LOADTEX_FORMAT_TEX

#define SXGC_LOADTEX_FORMAT_TEX   D3DFMT_FROM_FILE

#include <sxgcore.h>

формат в который будут преобразованы загружаемые текстуры

Предупреждения
если установить формат со сжатием то необходимо следовать всем правилам оформления текстур для данного формата

◆ LoadTexType

enum LoadTexType

#include <sxgcore.h>

типы текстур

Элементы перечислений
ltt_load 

загружаемая

ltt_const 

неудаляемая загружаемая

ltt_custom 

созданная пользователем

ltt_self 

самоопределение типа, на тот случай когда обновляем текстуру которая точно есть.

Если определить этот тип, а внутри у текстуры на самом деле нет типа (скорее всего нет текстуры) то будет определен ltt_load

создать место для текстуры tex и присвоить ей имя name, возвращает id.

Предупреждения
создавать текстур необходимо в managed pool (D3DPOOL_MANAGED) ибо обработка потери и восстановления устройства сюда не приходит

◆ SGCore_LoadTexUpdateN()

SX_LIB_API ID SGCore_LoadTexUpdateN ( const char *  name,
LoadTexType  type 
)

#include <sxgcore.h>

обновить/перезагрузить текстуру name, если текстуры не было в списке то добавляет.

Если текстуру надо обновить, но тип у нее заранее не известен, но она точно уже загружена то можно использовать тип самоопределения ltt_self, тогда тип текстуры не изменится
